Mimo [The One Billion Rows Challenge](/md/1brc.md) som riešil aj iné optimalizácie,
ktorá síce fungujú, ale ~~doma~~ v práci ich radšej neskúšajte, kolegovia vás za ne zabijú.

## Unrolling cyklov
Prvou optimalizáciou je unrolling for cyklov.
Namiesto klasického for-u sa použije to, že sa viacej prvok spracuje v jednom cykle,
najčastejšie sa to robí po štyroch.

```cs
for (int i = 0; i < buffer.Length; i++)
{
    buffer[i] = buffer[i] * a + c;
}
```
Sa zmení na:
```cs
for (int i = 0; i < buffer.Length; i += 4)
{
    buffer[i] = buffer[i] * a + c;
    buffer[i + 1] = buffer[i + 1] * a + c;
    buffer[i + 2] = buffer[i + 2] * a + c;
    buffer[i + 3] = buffer[i + 3] * a + c;
}
```

## Polia namiesto tried
Táto optimalizácia je vlastne proti tomu, ako by sa malo programovať keď pomocou štruktúr a tried enkapsulujeme dáta do logických celkov.
Pri tejto optimalizácii sa namiesto poľa štruktúr použijú viaceré polia zodpovedajúce fieldom pôvodnej štruktúry.
Takto je k nim pamäťovo lepší prístup. V novších .NET je k tomu možné výhodne použiť vektorizáciu.

Na toto existuje už hotový source generátor:
<https://github.com/Cysharp/StructureOfArraysGenerator>

## Modulo nahradiť if-om
V prípadoch, keď sa používa circular buffer, alebo niekde počítame často modulo, je ho lepšie hradiť obyčajným porovnaním a odčítaním, alebo bitovou maskou a následným dočítaním.

## Lock vs. Interlocked
Sú prípady, kedy ide nahradiť lokovanie pomocou triedy `Interlocked`, napríklad v príade cache alebo object poolu.
Viackrát som už refaktoroval zbytočné locky, niekedy to ide prepísať tak,
že sa 99% príadov vyrieši pomocou `Interlocked` a zvyšok pomocou locku.

Odporúčam pozornosť zamerať na metódu `Interlocked.CompareExchange()`.

## Eliminácia bound-check v poliach
V časti mojej diplomovej práci som používal viterbiho algoritmus a skryté markovove modely
na určovanie slovných druhov v slovenských vetách. Funguje to tak,
že máme skryty  markovov model, v ktorom sú zahrnuté pravdepodobnosti,
že jeden slovný druh nasleduje za druhým a pravdepodobnosti, že slovo vo vete patrí nejakému slovnému druhu
(ak slovo nebolo v trénovacích dátach, tak sa pravdepodobnosti odhadovali podľa prípony).

V čase písania diplomovej práce mi kompletné pretestovanie na testovacom korpuse trvalo cez osem hodín.
A väčšinu času zabralo práve vyhodnocovanie slovných druhov. K optimalizácii som sa dostal až neskôr.

Ešte upozorním na jednu zaujímavosť – v škole nás učili, že Slovenčina má desať slovných druhov,
no v skutočnosti ich máme jedenásť (jedenáste je _zvratané príčastie_),
ale v mojom riešení som ich požil dvanásť (u mňa bol dvanásty interpunkcia – bodky, čiarky, otázniky,...).

Viterbiho algoritmus patrí do algoritmov dynamického programovania,
kde sa pracuje z dvojrozmerným polom _NxM_, kde _N_ je počet slovných druhov, _M_ je dĺžka vety.
Celková zložitosť je _MxN^2^_. Jediné čo sa dalo optimalizovať sú alokácie a prístup k poliam.

V tomto algoritme sa násobia pravdepodobnosti, ale pri implementácii s násobením po niekoľkých
násobení pravdepodobností double podtečie.
Preto, je lepšie násobenie vymeniť za sčítanie logaritmov,
lebo platí: $ln(a \cdot b) = ln(a) + ln(b)$ a taktiež $ln(a) \geq ln(b)$ potom platí aj  $a \geq b$.

Ešte počas písania diplomovej práce som skúšal použiť unsafe kód a alokovať si polia pomocou `Marshal.AllocHGlobal()`, ale to rýchlosti nepomohlo.

Dnes už viem, že základom optimalizácie by bol `ArrayPool` pre alokovanie týchto polí.

Pri tomto riešení som mal veľkú výhodu, že síce bolo potrebné iterovať cez každé slovo,
ale vnútorné cykly iterujú nad kombináciou konkrétneho slova a slovného druhu.
Takže cyklus šiel rozbaliť, čo unieslo nejakú úsporu času (testovanie podmienky vo _for_).
Keď som daný cyklus otočil a namiesto indexov 0 až 11 som šiel od 11 k 0, tak som ušetril viac času,
lebo kompilátor usúdil, že hranice poľa treba testovať len na prvom jedenástom indexe.


```cs
int T = this.words.Length;
double** a = allocmatrixDouble(this.N, T);
try
{
    for (int i = 0; i < this.N; i++)
    {
        a[i][0] = -1.0 * this.beginStates[i] - this.GetWordPropability(i, this.words[0]);
    }

    double minWeight = 0.0, weight;
    for (int t = 1; t < T; t++)
    {
        for (int j = 0; j < this.N; j++)
        {
            minWeight = a[0][t - 1] - this.pseudoTransmision[0];

            for (int i = 1; i < this.N; i++)
            {
                weight = a[i][t - 1] - this.pseudoTransmision[i];

                if (weight < minWeight)
                {
                    minWeight = weight;
                }
            }
            a[j][t] = minWeight - this.GetWordPropability(j, this.words[t]);
        }
    }

    return minWeight;
}
finally
{
    freeMatrix(a, T);
}
```
